什么是fiber

fiber 是react 的最小工作单元，在react 的世界中 一切都可以组件。每一个组件对应一个fiber节点，许多fiber 节点相互嵌套，关联，就组成了fiber 树

一个dom 节点对应一个fiber 节点，但一个fiber节点不一定有对应的dom 节点

fiber架构下的react是如何更新的

react 更新分为两个阶段，第一个阶段是render阶段 第二个阶段是commit阶段，分别是fiber 树的构建和更新最终效果的应用

render阶段

render阶段实际上是内存中构建一棵新的fiber树（成为workinProgress树）,构建过程是依照现有的fiber树从root开始深度优先遍历再回溯到root的过程,这个过程中每个fiber节点都会经历两个阶段：beginWork 和completeWork.

组件的状态计算，diff的操作以及render函数的执行，发生在beginWork阶段，effect链表的收集。被跳过的优先级的收集，发生在completeWork阶段。构建workinProgress树的过程中会有一个workInProgress的指针记录当下构建到哪个fiber节点,这是react 更新任务可恢复的重要原因之一。

commit阶段

在render阶段结束后，会进入commit阶段，该阶段不可中断，主要依据workinprogress树种有哪些变化的节点，去完成dom 操作，将应用更新在页面上，除此之外，还会异步调度useEffect以及同步执行useLayoutEffct

scheduler的作用

scheduler根据任务优先级来决定哪个任务先被执行，保证最高优先级的任务最先被执行

scheduler根据时间片去终止任务，并判断任务是否完成，若未完成 则继续完成任务函数，他只是去做任务的恢复和中断，而任务是否完成需要依赖react 告诉它，scheduler和react 相互匹配的模式 可以让react 的任务执行具备异步可中断的特点。
